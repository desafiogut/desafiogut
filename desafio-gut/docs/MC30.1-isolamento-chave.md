# MC30.1 — Isolamento da Chave Mestra (OpenZeppelin Defender Relay)

> PILAR 2 (EVERYTHING CLOUD CODE): documentação modular da arquitetura de
> assinatura isolada. Branch `feat/mc30.1`. Ver também `security_audit.md` §MC30.1.

## Objetivo

Eliminar a chave privada bruta (`COORDENACAO_PRIVATE_KEY`) do ambiente serverless
de mainnet, migrando a assinatura de transações da coordenação para o **HSM** do
OpenZeppelin Defender Relay — acessível apenas via credenciais de API escopadas e
revogáveis. O contrato `Leilao.sol` **não é alterado** (R10): o Relayer assina como
o próprio EOA da coordenação.

## Arquitetura

```
                         ┌─────────────────────────────┐
  contract.mjs ─┐        │      _lib/signer.mjs        │
  consolidar-   ├──────► │   (FACHADA DE ASSINATURA)   │
   lances.mjs   │        │  backendAssinatura()        │
  ia-preditiva  ┘        │  obterSignerCoordenacao()   │
                         └───────────┬─────────────────┘
                                     │ seleção por env
                  ┌──────────────────┴───────────────────┐
            local-key (testnet)                    defender (mainnet)
         new ethers.Wallet(chave)            DefenderRelayProvider +
         provider = JsonRpcProvider          DefenderRelaySigner (HSM)
         RPC_URL / CONSOLIDATION_RPC_URL      DEFENDER_API_KEY/SECRET
```

- **Antes:** `new ethers.Wallet(COORDENACAO_PRIVATE_KEY)` em 3 ficheiros.
- **Depois:** `new Wallet(...)` existe **apenas** em `_lib/signer.mjs`; os 3
  call-sites obtêm o signer via `obterSignerCoordenacao(rpcUrl)`.

## Seleção de backend

| `SIGNER_BACKEND` | `NETWORK_STAGE` | Backend ativo |
|------------------|-----------------|---------------|
| `defender`       | (qualquer)      | `defender`    |
| `local-key`      | (qualquer)      | `local-key`   |
| (vazio)          | `mainnet`       | `defender`    |
| (vazio)          | sepolia/outro   | `local-key`   |

## API da fachada (`_lib/signer.mjs`)

| Export | Descrição |
|--------|-----------|
| `obterSignerCoordenacao(rpcUrl)` | `async` → `{ provider, signer, address, backend }`. Signer compatível com ethers v6 (passa a `new Contract(...)`, expõe `signTypedData`). |
| `backendAssinatura()` | `'defender' \| 'local-key'`. |
| `resolverChaveCoordenacao()` | Chave bruta do env (canónica + fallback `COORDENACAO_PRIVATE`), ou `null`. |
| `getCoordenacaoAddressCache()` | Endereço resolvido na última assinatura (síncrono). |
| `assertChaveBrutaAusenteEmMainnet()` | Guarda: lança se mainnet + chave bruta presente (ITEM 3.5). |

## Variáveis de ambiente

```
NETWORK_STAGE=sepolia|mainnet
SIGNER_BACKEND=defender|local-key      # opcional, sobrepõe a heurística
DEFENDER_API_KEY=...                    # mainnet (HSM)
DEFENDER_API_SECRET=...                 # mainnet (HSM)
DEFENDER_RELAYER_ADDRESS=0x...          # opcional (evita 1 round-trip)
COORDENACAO_PRIVATE_KEY=0x...           # SÓ testnet (backend local-key); proibida em mainnet
```

Ver `frontend/.env.example` para descrições completas.

## Pacote

`@openzeppelin/defender-sdk` (Ethers v6). O `@openzeppelin/defender-relay-client`
do enunciado original está **DEPRECATED** (era Ethers v5) — substituído pelo SDK
mantido, que declara `ethers ^6.9.0`.

## Cutover (ações humanas — ver `security_audit.md` §MC30.1.5)

1. Criar Relayer no Defender + obter credenciais.
2. Validar handshake real.
3. Transferir coordenação on-chain (two-step) e confirmar.
4. Remover a chave bruta do env de mainnet do Netlify.
5. (Opcional) Gnosis Safe multisig.
