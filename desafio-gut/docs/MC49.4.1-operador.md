# MC49.4.1 — Instrução ao Operador (configuração da chave)

> **A chave privada NUNCA passa pelo chat nem por CLI do agente (R9/R10).**
> Todos os comandos abaixo são executados **pelo operador**, na própria shell.

## Carteira alvo (confirmada on-chain)

A `COORDENACAO_PRIVATE_KEY` deve ser a chave privada da carteira que é a
`coordenacao()` do contrato **live** `0x59A73Acc8E8B210C874B0E3A9eC9B8B64847F6D5`:

```
Carteira de coordenação = 0xdEbe637d7f74C4bfe71263920F68589f0c672D92
```

Se configurar a chave de outra carteira, a compra falhará com
`"wallet X não é coordenacao do contrato Y"` (não com o erro atual).

## Passo 1 — (recomendado) Confirmar que a sua chave local bate com a carteira alvo

O `.env.local` já contém um `COORDENACAO_PRIVATE_KEY`. Confirme que ele deriva
para `0xdEbe637d…2D92` **sem imprimir a chave** (imprime só o endereço):

```bash
cd desafio-gut/frontend
node -e "import('ethers').then(async ({Wallet})=>{const fs=await import('fs');const m=fs.readFileSync('.env.local','utf8').match(/^COORDENACAO_PRIVATE_KEY=(.+)$/m);if(!m){console.log('sem chave em .env.local');return;}console.log('endereco da chave:', new Wallet(m[1].trim()).address);})"
```

Esperado: `endereco da chave: 0xdEbe637d7f74C4bfe71263920F68589f0c672D92`.
Se **não** bater, NÃO configure — traga a chave correta da carteira `0xdEbe637d…2D92`.

## Passo 2 — (sanity opcional) Confirmar o contrato do backend

`CONTRATO_SEPOLIA` é público (listado em `SECRETS_SCAN_OMIT_KEYS`). Confirme que o
backend credita no mesmo contrato que a UI lê:

```bash
netlify env:get CONTRATO_SEPOLIA --context production
```

Esperado: `0x59A73Acc8E8B210C874B0E3A9eC9B8B64847F6D5` (ou vazio → usa
`VITE_CONTRATO_SEPOLIA`, que é o mesmo). Se for `0x273Ef96f…445e`, avise o agente
(a carteira de coordenação seria outra: `0xDa3a83…e84E`).

## Passo 3 — Configurar a chave no Netlify (produção)

```bash
netlify env:set COORDENACAO_PRIVATE_KEY "0x<CHAVE_DA_CARTEIRA_0xdEbe637d>" --context production
```

- NÃO partilhe a chave no chat.
- `NETWORK_STAGE` permanece Sepolia (não definir `mainnet` — R3 / mainnet-readiness).
  O backend `local-key` é o correto para testnet.

## Passo 4 — Redeploy (as functions releem o env só num novo deploy)

Opção A (dashboard): **Clear cache and deploy site**.
Opção B (CLI):

```bash
netlify deploy --build --prod --message "chore(mc49.4.1): configurar COORDENACAO_PRIVATE_KEY"
```

## Passo 5 — Avisar o agente

Após o deploy ficar `ready`, avise o agente. Ele valida:

```
GET /.netlify/functions/health  →  SIGNER_READY:"set"  (era "MISSING")
```

E depois prossegue para a validação visual (Segmento 3), que exige o seu login
manual (Privy OAuth não é automatizável).
